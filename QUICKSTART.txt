Follow these simple steps to get your bot running in minutes!

================================================================================
STEP 1: PREREQUISITES
================================================================================

Make sure you have:
‚úì Linux VPS with Ubuntu 20.04+
‚úì Docker installed
‚úì OpenAI or Anthropic API key
‚úì Telegram bot token (from @BotFather)
‚úì Kasioon.com API credentials (TODO: will be provided)
‚úì WhatsApp Business API access (optional)

================================================================================
STEP 2: CONFIGURE ENVIRONMENT
================================================================================

1. Navigate to project directory:
   cd /var/www/html/kasioon-bot

2. Copy environment template:
   cp env-config.txt .env

3. Edit configuration (IMPORTANT!):
   nano .env

   Required settings:
   - PORT=3355 (already configured)
   - OPENAI_API_KEY=sk-your-key-here (OR ANTHROPIC_API_KEY)
   - TELEGRAM_BOT_TOKEN=your-telegram-token
   - KASIOON_API_URL=https://api.kasioon.com:3850 (TODO: update with actual API)
   - KASIOON_API_KEY=your-api-key (TODO: update when provided)

   Optional but recommended:
   - WHATSAPP_API_TOKEN=your-whatsapp-token
   - N8N_USER=your-username
   - N8N_PASSWORD=strong-password

   Save and exit (Ctrl+X, Y, Enter)

================================================================================
STEP 3: DEPLOY
================================================================================

Run the deployment script:

   chmod +x deploy.sh
   ./deploy.sh

This will:
- Create necessary directories
- Build and start all Docker containers
- Check service health

Wait 30-60 seconds for all services to start.

================================================================================
STEP 4: VERIFY DEPLOYMENT
================================================================================

Check if services are running:

   docker-compose ps

All services should show "Up" status.

Test API health:

   curl http://localhost:3355/health

You should see: {"status":"ok",...}

================================================================================
STEP 5: CONFIGURE TELEGRAM
================================================================================

Option A: Using the setup script (easiest):

   chmod +x setup-telegram.sh
   ./setup-telegram.sh

   Enter your webhook URL when prompted:
   https://your-domain.com/api/telegram/webhook

Option B: Manual setup:

   curl -X POST "https://api.telegram.org/bot<YOUR_TOKEN>/setWebhook" \
     -d "url=https://your-domain.com/api/telegram/webhook"

STEP 7: CONFIGURE WHATSAPP (OPTIONAL)
================================================================================

1. Go to Meta Business Manager
2. Navigate to WhatsApp > Configuration
3. Add webhook URL: https://your-domain.com/api/whatsapp/webhook
4. Add verify token (from your .env file)
5. Subscribe to "messages" event
6. Test by sending a message

================================================================================

4. Configure and activate workflows

================================================================================
TROUBLESHOOTING
================================================================================

‚ùå Services won't start:
   docker-compose logs

‚ùå Bot not responding:
   docker-compose logs -f api

‚ùå Can't access from internet:
   - Check firewall: sudo ufw allow 80
   - Check domain DNS settings
   - Verify webhook URL is correct

‚ùå AI not working:
   - Check API key in .env
   - Verify API provider (openai/anthropic)
   - Check logs: docker-compose logs -f api
